### Next.js + Drizzle ORM

**Step 4 content:**
```markdown
## Step 4: Architecture Review — Database & API Routes

- [ ] `src/db/schema.ts` — any schema changes properly migrated?
  - Use `/schema-migrate` to analyze schema changes before pushing:
    ```bash
    /schema-migrate
    ```
  - Then push with appropriate commands:
    ```bash
    npm run db:push
    npm run db:generate
    ```
- [ ] `src/lib/adapters/` — any adapter changes backwards-compatible?
- [ ] `src/app/api/` — new routes handle errors + auth correctly?
- [ ] `src/app/actions/` — server actions properly typed?
```

**Step 5 content:**
```markdown
## Step 5: Code Quality Checklist

- [ ] TypeScript strict mode: `npm run lint` passes
- [ ] No missing types or `any`
- [ ] API routes have error handling
- [ ] Database queries optimized (no N+1 in loops)
- [ ] Environment variables documented
```

**Architecture Notes:**
```
- **Schema**: `src/db/schema.ts` with Drizzle ORM, SQLite backend
- **Migrations**: Generated by `npm run db:generate`, applied with `npm run db:push`
- **Schema safety**: Always run `/schema-migrate` before schema changes
- **API Routes**: Centralized in `src/app/api/`, use server actions for mutations
- **Business Logic**: Adapters in `src/lib/adapters/`, services in `src/lib/`
```

### Laravel + Eloquent ORM (MySQL/Postgres/SQLite)

**Step 4 content:**
```markdown
## Step 4: Architecture Review — Database & Migrations

- [ ] `database/migrations/` — any new migrations?
  - Use `/schema-migrate` to analyze schema changes:
    ```bash
    php artisan make:migration [description]
    php artisan migrate
    ```
  - Rollback if needed:
    ```bash
    php artisan migrate:rollback --step=1
    ```
- [ ] `app/Models/` — any new Eloquent models or relationships?
  - Verify relations match database foreign keys
  - Check `$fillable` / `$guarded` for mass assignment safety
- [ ] `database/seeders/` — any seeders for new data?
- [ ] `routes/` — new endpoints have proper auth + validation?
```

**Step 5 content:**
```markdown
## Step 5: Code Quality Checklist

- [ ] Migrations follow naming convention: `YYYY_MM_DD_HHMMSS_action.php`
- [ ] Migration `up()` and `down()` are complete and reversible
- [ ] Eloquent models have proper type hints on relations
- [ ] Database queries optimized (use `with()` to prevent N+1)
- [ ] Input validation on all routes (Form Requests or Validator)
- [ ] Mass assignment protection ($fillable / $guarded set correctly)
- [ ] Tests pass: `php artisan test`
```

**Architecture Notes:**
```
- **Database**: MySQL/Postgres/SQLite via Eloquent ORM (configured in .env)
  - MySQL: DB_CONNECTION=mysql — supports online DDL (MySQL 8.0+)
  - Postgres: DB_CONNECTION=pgsql — full ALTER TABLE support
  - SQLite: DB_CONNECTION=sqlite — limited ALTER, use /schema-migrate for complex changes
- **Migrations**: `php artisan make:migration` → `php artisan migrate` → rollback with `php artisan migrate:rollback`
- **Schema safety**: Always run `/schema-migrate` before migrations (especially critical for SQLite)
- **Models**: Eloquent in `app/Models/`, use relationships for FK traversal
- **Validation**: Form Requests in `app/Http/Requests/`
- **Testing**: Feature tests in `tests/Feature/`, unit tests in `tests/Unit/`
```
