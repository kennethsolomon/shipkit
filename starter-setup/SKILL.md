## Overview

Creates an optimized CLAUDE.md file for new or existing projects by auto-detecting the tech stack and project details.

### What it does:
1. Detects project language (JavaScript, Python, Go, Rust)
2. Identifies framework (React, Django, FastAPI, etc.)
3. Finds database setup (Prisma, SQLAlchemy, etc.)
4. Extracts build/test/lint commands
5. Generates optimized CLAUDE.md (< 150 lines)
6. Protects custom files via sidecar creation

### Key Features:
- **Auto-detection**: Analyzes package.json, pyproject.toml, Cargo.toml, etc.
- **Structure Discovery**: Auto-discovers actual project directories (src/, tests/, docs/, etc.)
- **Documentation Discovery**: Finds and links documentation files (README, CONTRIBUTING, docs/)
- **Workflow Discovery**: Detects npm scripts, Makefile targets, GitHub Actions workflows
- **Template-driven**: Uses optimized template with all essential sections
- **Tailored Output**: Generated CLAUDE.md includes discovered sections specific to your project
- **Safe updates**: Detects custom files and writes suggestions instead
- **Marker-based**: Uses `<!-- Generated by /setup-claude-tools -->` for safe regeneration
- **Maintainable**: Output ranges from ~100 lines for small projects to 150+ for projects with discovered structure

## Usage

```bash
# Run from project root to create CLAUDE.md
/setup-starter
```

## Output

Creates either:
- **CLAUDE.md** - If no custom file exists
- **CLAUDE.md.setup-claude.md** - If custom CLAUDE.md exists (review and merge)

## What's Included

Each generated CLAUDE.md includes:

### Template Sections (Always)
- Project name and description
- Technology stack (language, framework, database, UI, testing)
- Quick start command
- Development workflow (setup, run, test, lint commands)
- Build commands
- Deployment notes
- Environment variables guide

### Discovered Sections (When Found)
- **Key Directories** - Auto-discovered: src/, tests/, docs/, config/, etc.
- **Documentation & Resources** - Auto-discovered: README.md, CONTRIBUTING.md, docs/*.md
- **Common Workflows** - Auto-discovered: npm scripts, Makefile targets, GitHub Actions

### Example Output

For a React TypeScript project, you'll see:
```
✅ CLAUDE.md created successfully!
   Lines: 95 (25 lines added from discoveries)

📁 Discoveries from project structure:
   📂 4 directories found (src/, tests/, docs/, config/)
   📚 3 documentation files (README.md, CONTRIBUTING.md, docs/API.md)
   🔧 Workflows discovered and documented
```

2. Customize as needed (project-specific details)
3. Run `/claude-doctor` to check for issues
4. Run `/optimize-claude` if file exceeds 150 lines
