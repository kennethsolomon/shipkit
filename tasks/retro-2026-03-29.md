## What Went Well

- **Parallel audit coverage was thorough** — 3 Explore agents covering 43 skills + 13 agents + 13 commands simultaneously found issues that a sequential scan would have missed or taken much longer to surface
- **All 43 skills came back clean** — zero frontmatter issues, zero broken agent references, zero stale step numbers in skill files themselves
- **Critical template gap caught** — the setup-claude template missing steps 5.5/8.5/8.6 would have silently shipped broken workflows to every new project bootstrapped after v3.17.x
- **Maintenance guide created** — future workflow changes now have a concrete checklist instead of relying on memory

## What Didn't Go Well

- **Duplicate release commits** — combined the release script and git push in one command (`printf... | bash release.sh; git push`), causing the script to run twice. Produced two "chore: Release v3.18.0" commits in the log. Minor but messy.
- **Agent files are gitignored** — the DESIGN NOTE comments added to debugger, qa-engineer, and tech-writer live on disk but won't be tracked in the repo. Anyone running /sk:setup-claude on a new machine gets agents without the design rationale. The notes need to live in the templates instead.

## Patterns

- **Setup-claude template divergence is a recurring risk** — this is the second time a workflow update was made to CLAUDE.md but not propagated to the template. The maintenance guide now makes this explicit, but the real fix would be a lint check in CI.
- **Read-before-edit discipline** — one edit failed because the file wasn't read first. The Bash tool requires a prior Read. This is a consistent constraint to internalize.

## Action Items

2. **Never combine release script + git push in one command** — always run `printf "..." | bash release.sh` first, verify the output, then push separately. Apply: every release.

3. **Add a template-sync check to /sk:setup-optimizer** — the optimizer already checks CLAUDE.md workflow against the canonical 11-step definition. It should also verify the setup-claude template matches. Apply: next sk:setup-optimizer improvement session.
