---

## Overview

`/sk:website` builds a complete, client-deliverable multi-page marketing website from a brief, URL, or one sentence. It runs autonomously from intake to handoff — extracting business facts, generating real copy, applying art direction, injecting WhatsApp CTAs for local businesses in PH/SEA, enforcing Lighthouse 90+ quality, and producing a full client handoff package (HANDOFF.md, DEPLOY.md, CONTENT-GUIDE.md).

This is distinct from `/sk:mvp`, which generates a validation prototype with fake data. `/sk:website` produces a site you hand to a client.

## Business Logic

1. **Brief extraction** — accepts Google Maps URL, existing site URL, or plain text; auto-advances without approval unless facts are contradictory
2. **Parallel research** — launches 3 agents simultaneously: Strategy (sitemap + sections), Copy (all page copy using real facts only), Art Direction (aesthetic direction + palette + typography)
3. **Build** — implements full site using all 3 agent outputs; real copy injected at build time; no placeholders
4. **WhatsApp CTA** — detects PH/SEA local business from location signals; injects floating button; uses Messenger alternative if preferred
5. **Lighthouse enforcement** — if Playwright MCP available, runs Lighthouse on each page and loops until 90+ on Performance, Accessibility, SEO, Best Practices; falls back to static quality pass otherwise
6. **Launch audit** — runs 5-category checklist from `references/launch-checklist.md`; fixes blockers
7. **Handoff package** — generates 3 client files from templates with real project values substituted

---

## Hard Rules

- Real copy always — never Lorem ipsum, never `[placeholder]` headlines
- Multi-page by default — Home, About, Services/Menu, Contact + niche extras
- Default stack: Next.js + Tailwind (respects existing project stack)
- WhatsApp is default contact for local businesses in PH/SEA
- Lighthouse 90+ required before handoff (loops until passing)
- Never invent business facts — no fake testimonials, invented certifications, or made-up outcomes
- Parallel agents for speed — strategy + copy + art direction run simultaneously
- Revision mode is targeted — never full rebuild for small changes

## Edge Cases

| Scenario | Behavior |
|---|---|
| Google Maps URL fails to fetch | Falls back to plain-text extraction immediately |
| Business type not in niche table | Uses `references/niche/local-business.md` as default |
| Location not specified | Injects WhatsApp with `[PHONE]` placeholder, notes in HANDOFF.md |
| Playwright MCP not available | Falls back to static quality pass (build check + manual SEO/a11y verification) |
| User provides minimal 1-sentence brief | Infers missing fields; confirms in summary before building |
| `--revise` flag with vague feedback | Applies best interpretation; summarizes changes made |
| Existing project with different stack | Works within existing stack, does not scaffold Next.js |

---

## Error States

| Condition | Behavior |
|---|---|
| WebFetch fails on URL | Falls back to Option B (plain text extraction) |
| One parallel agent errors | Logs error; retries once; if still failing, proceeds with available outputs |
| Lighthouse fails to reach 90+ after 3 iterations | Reports specific failing audits to user; proceeds to handoff |
| Build fails (`next build` error) | Fixes build errors before Lighthouse; does not skip |
