## Overview

Creates an optimized CLAUDE.md file for new or existing projects by auto-detecting the tech stack and project details.

### What it does:
1. Detects project language (JavaScript, Python, Go, Rust)
2. Identifies framework (React, Django, FastAPI, etc.)
3. Finds database setup (Prisma, SQLAlchemy, etc.)
4. Extracts build/test/lint commands
5. Generates optimized CLAUDE.md (< 150 lines)
6. Protects custom files via sidecar creation

### Key Features:
- **Auto-detection**: Analyzes package.json, pyproject.toml, Cargo.toml, etc.
- **Template-driven**: Uses optimized template with all essential sections
- **Safe updates**: Detects custom files and writes suggestions instead
- **Marker-based**: Uses `<!-- Generated by /setup-claude-tools -->` for safe regeneration
- **Line optimization**: Keeps files under 150 lines for efficiency

## Usage

```bash
# Run from project root to create CLAUDE.md
/setup-starter
```

## Output

Creates either:
- **CLAUDE.md** - If no custom file exists
- **CLAUDE.md.setup-claude.md** - If custom CLAUDE.md exists (review and merge)

## What's Included

Each generated CLAUDE.md has:
- Project name and description
- Technology stack (language, framework, database, UI, testing)
- Quick start command
- Project structure
- Development workflow
- Build commands
- Deployment notes
- Environment variables guide
- Documentation links

## Next Steps

1. Review the generated CLAUDE.md
2. Customize as needed (project-specific details)
3. Run `/claude-doctor` to check for issues
4. Run `/optimize-claude` if file exceeds 150 lines
